# TrekMate EXE Backend

Minimal Spring Boot backend serving the 4 tour execution APIs for the TrekMate Android app.

## Prerequisites

- Java 21
- Maven 3.9+
- PostgreSQL running locally

## Database Setup

Create the database before starting:

```sql
CREATE DATABASE exetrekmate;
```

Connection defaults (configured in `application.yml`):
- Host: `localhost:5432`
- Database: `exetrekmate`
- Username: `postgres`
- Password: `123456`

Tables are auto-created by Hibernate (`ddl-auto: update`) on first startup.

## Run

```bash
cd exe-backend
mvn spring-boot:run
```

Server starts on **port 8081**.

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/exe/tours` | Create a new tour (leader) |
| `POST` | `/exe/tours/join` | Join a tour (member) |
| `POST` | `/exe/tours/end` | End a tour (leader only) |
| `GET`  | `/exe/tours/{tourId}/members` | Get member list |

## Swagger UI

Open in browser after startup: http://localhost:8081/swagger-ui.html

## Android Connection

The Android app (emulator) connects to this backend via:
```
http://10.0.2.2:8081/
```

For physical device on same network:
```
http://<your-local-ip>:8081/
```

Update `BASE_URL` in `app/build.gradle.kts` → debug block accordingly.

## ID Format Notes

- `leader_id` / `user_id`: 16-char compact hex string generated by Android `UserIdGenerator`
- `group_id`: 8-char uppercase alphanumeric — sized to fit the BLE advertisement packet's 8-byte groupId field
- `join_code`: 6-char uppercase alphanumeric — displayed on screen for manual entry
- `qr_payload`: `trekmate://join?code=<joinCode>` — scanned via QR camera
